# TCGspot 上架前測試清單

> 用法:在**真機**上裝**release 版**,用**全新帳號**從頭走一遍。每條測完打勾 ✅。
> 重點:不要只用開發帳號/模擬器/debug 版測 —— 真實用戶拿到的是 release 版 + 真手機 + 空白新帳號。

---

## 0. 測試環境準備(先做)
- [ ] 用 release build 測:`flutter build apk --release --dart-define-from-file=env.json`(Android)
- [ ] 至少一支**真 Android** + 一支**真 iPhone**(或 TestFlight / Play 內部測試)
- [ ] 準備 **2 個全新帳號**(A、B)互相操作
- [ ] 其中一次用**全新安裝 + 全新帳號**(模擬第一次的用戶)
- [ ] 已裝 Crashlytics(上架後當機回報)— 建議

---

## 1. 安裝 / 首次啟動
- [ ] 全新安裝後能正常開啟,不閃退
- [ ] 開機畫面顯示 **TCGspot**(非 pokebid)
- [ ] 桌面圖示名稱是 **TCGspot**
- [ ] 通知權限彈窗正常出現,允許/拒絕都不會壞
- [ ] 首次進入不會卡白畫面

## 2. 登入 / 帳號
- [ ] Google 登入成功(真機)
- [ ] **Sign in with Apple** 登入成功(iOS,設定完成後)
- [ ] iOS 上 Apple 按鈕有顯示;Android 只顯示 Google
- [ ] 「先瀏覽不登入」可用,未登入時受限功能會提示登入
- [ ] 登出正常
- [ ] **刪除帳號**:刪完資料清乾淨,重新登入是全新帳號(無殘留舊資料)
- [ ] 登入後 `profiles.fcm_token` 有寫入

## 3. 掛售(刊登)
- [ ] 能建立掛售(直購 / 競標)
- [ ] 上傳卡片照片成功(多張)
- [ ] 選卡、填價格、分級、描述都正常
- [ ] 沒填必填欄位有提示
- [ ] 掛售後出現在列表/個人檔案

## 4. 瀏覽 / 搜尋 / 篩選
- [ ] 首頁卡片正常載入
- [ ] 搜尋卡名有結果
- [ ] 篩選條件正常
- [ ] **沒圖片的卡顯示「暫時未提供該卡圖」**(不是破圖)
- [ ] 商品詳情頁正常(多圖切換、放大)

## 5. 出價流程
- [ ] B 能對 A 的掛售出價
- [ ] A 收到出價通知(站內 + 推送)
- [ ] 接受 / 拒絕出價正常
- [ ] 出價金額驗證(不能 0 或負數)

## 6. 聊天
- [ ] A、B 能互傳訊息
- [ ] 訊息即時更新
- [ ] 收訊息方收到推送(對方在背景時)
- [ ] 圖片/長訊息顯示正常

## 7. 推送通知(逐項)
> App 切**背景**測。每類型觸發一次,確認會跳 + 點擊跳到正確畫面。
- [ ] 收到**出價**通知 → 點擊跳商品頁
- [ ] 收到**訊息**通知 → 點擊跳聊天
- [ ] 收到**評價**通知
- [ ] 收到**願望清單到貨**通知
- [ ] 通知會**彈出橫幅**(不只進通知欄)
- [ ] App 完全關閉時仍收得到
- [ ] iOS 上推送正常(APNs 設定完成後)

## 8. 評價
- [ ] 交易後能留評價
- [ ] 評價顯示在對方個人檔案
- [ ] 不能重複/灌假評價

## 9. 圖鑑 / 收藏 / 願望清單
- [ ] 圖鑑系列、卡片正常瀏覽
- [ ] 系列 logo 沒圖時有佔位顯示
- [ ] 加入/移除收藏正常
- [ ] 收藏市值/盈虧計算正確
- [ ] 願望清單加入後,有符合的掛售會通知

## 10. 安全(UGC,App Store 必查)
- [ ] 能**檢舉**不當內容/用戶
- [ ] 能**封鎖**用戶,封鎖後看不到對方內容/訊息
- [ ] 被封鎖方無法再聯絡
- [ ] 私隱政策、服務條款從設定頁可開啟

## 11. 資料隔離(RLS,很重要)
- [ ] A **看不到** B 的私人資料(如別人的草稿、未公開資訊)
- [ ] A **改不到 / 刪不到** B 的掛售、評價、訊息
- [ ] 未登入用戶不能做需登入的操作

## 12. 多語言
- [ ] 切換中文 / 英文,全畫面文字正確(無漏翻、無 PokeBid 殘留)
- [ ] 法律文件中英版都正常

## 13. 邊界 / 網路
- [ ] 慢速網路下不會卡死(有 loading)
- [ ] 離線時有合理提示,不閃退
- [ ] 空狀態畫面正常(沒掛售、沒通知、沒收藏)
- [ ] 長文字/特殊字元不會破版

---

## 14. 設定項確認(技術,上架前必修)
- [ ] Bundle ID 改成正式(非 `com.example.pokebid`)
- [ ] iOS / Android 顯示名稱 = TCGspot
- [ ] env.json 密鑰正確打包進 release(release 版連得到 Supabase)
- [ ] Sign in with Apple 後台設定完成(Service ID / Key / Supabase provider)
- [ ] iOS APNs 金鑰已上傳 Firebase
- [ ] 隱私政策 URL、服務條款 URL 已上線(suloktcg.com)

## 15. 上架資料(送審用)
- [ ] App Store / Play metadata 填好(描述、截圖、分類)
- [ ] 隱私政策 URL 填入
- [ ] **提供審核用 Demo 帳號**(審核員要能登入測試)
- [ ] 寶可夢 IP 聲明已備(降低 5.2 被拒風險)
- [ ] 年齡分級填寫

---

## 上架策略建議
1. 先發 **TestFlight / Play 內部測試**,自己 + 幾個人裝真 build 測
2. 照本清單跑一遍核心流程
3. Android 用**分階段放量**(5% → 20% → 100%)
4. Crashlytics 監控,出事即時修
